## Problem Statement
```
You are given three integers n, l, and r.

A ZigZag array of length n is defined as follows:

Each element lies in the range [l, r].
No two adjacent elements are equal.
No three consecutive elements form a strictly increasing or strictly decreasing sequence.
Return the total number of valid ZigZag arrays.

Since the answer may be large, return it modulo 109 + 7.

A sequence is said to be strictly increasing if each element is strictly greater than its previous one (if exists).

A sequence is said to be strictly decreasing if each element is strictly smaller than its previous one (if exists).
```

### Constraints:
```
3 <= n <= 2000
1 <= l < r <= 2000
```


## Approach-1: Recursive
- Find the total number of available values in the range [l, r]. 
- Consider every value as a possible starting element of the zig-zag array. 
- For each starting value:
  - Assume the next element must be greater and count all valid arrays. 
  - Assume the next element must be smaller and count all valid arrays.
- During recursion: 
  - Keep track of the current position, previous value, and required direction. 
  - If the required direction is increasing: Try every value greater than the previous value. 
  - If the required direction is decreasing: Try every value smaller than the previous value.
- After choosing a value:
  - Move to the next position. 
  - Reverse the direction (increase ↔ decrease).
- When the array length becomes n: One valid zig-zag array is formed.  Return 1.
- Sum all valid ways obtained from all starting values and directions.

---

### Code:
```java
class Solution {

    static int helper(int index,int prev,boolean isIncreasing,int m,int n){
        if(index==n){
            return 1;
        }

        int res=0;
        if(isIncreasing){
            for(int val=prev+1;val<=m;val++){
                res = (res + helper(index+1,val,false,m,n)) % 1000000007;
            }
        }
        else{
            for(int val=1;val<prev;val++){
                res = (res + helper(index+1,val,true,m,n)) % 1000000007;
            }
        }
        return res;
    }

    public int zigZagArrays(int n, int l, int r) {
        int res = 0;
        int m = r-l+1;
        for(int start=1;start<=m;start++){
            res = (res + helper(1,start,true,m,n)) % 1000000007;
            res = (res + helper(1,start,false,m,n))  % 1000000007 ;
        }
        return res;
    }
}
```

### Complexity
```
Let:
m = r - l + 1
At each level, up to m choices may be explored and recursion depth is n.

Recursion depth = n
```
- Time : O(m^(n+1))
- Space : O(n)

## Approach-2: Memoization
- Find the total number of available values in the range [l, r].
- Define a DP state using:
  - Current position in the array. 
  - Previous chosen value. 
  - Required direction (increase or decrease).
- For every possible starting value:
  - Start with increasing direction. 
  - Start with decreasing direction. 
  - Add both counts to the final answer.
- For each state:
  - If the current position reaches length n, a valid zig-zag array is formed, return 1.
- Before computing a state:
  - Check whether its answer is already stored in DP. 
  - If yes, return the stored value.
- Otherwise:
  - If the next element must be greater, try all values greater than the previous value. 
  - If the next element must be smaller, try all values smaller than the previous value. 
  - Recursively count valid arrays while alternating the direction. 
  - Store the computed answer in DP and return it. 
  - Sum all valid ways obtained from all starting values.

---

### Code:
```java
class Solution {

    static int[][][] arr;
    static int n, m;

    static int helper(int index, int prev, int isIncreasing) {
        if (index == n) {
            return 1;
        }

        if(arr[index][prev][isIncreasing]!=-1){
            return arr[index][prev][isIncreasing];
        }

        int res = 0;
        if (isIncreasing==1) {
            for (int val = prev + 1; val <= m; val++) {
                res = (res + helper(index + 1, val, 0)) % 1000000007;
            }
        } else {
            for (int val = 1; val < prev; val++) {
                res = (res + helper(index + 1, val, 1)) % 1000000007;
            }
        }
        return arr[index][prev][isIncreasing]=res;
    }

    public int zigZagArrays(int N, int l, int r) {
        int res = 0;
        m = r - l + 1;
        n = N;

        arr = new int[n][m + 1][2];

        for (int i = 0; i < arr.length; i++) {
            for (int j = 0; j < arr[i].length; j++) {
                for (int k = 0; k < 2; k++) {
                    arr[i][j][k]=-1;
                }
            }
        }

        for (int start = 1; start <= m; start++) {
            res = (res + helper(1, start, 1)) % 1000000007;
            res = (res + helper(1, start, 0)) % 1000000007;
        }
        return res;
    }
}
```

### Complexity
```
Time Complexity
Total states = n × m × 2
For each state, up to m transitions are explored.

Time: O(n × m²)

where m = r - l + 1
```

```
Space Complexity
DP table: O(n × m × 2)
Recursion stack: O(n)

Space: O(n × m)
```

## Approach-3: Tabulation

### Code:
```java
class Solution {
    public int zigZagArrays(int n, int l, int r) {
        int m = r - l + 1;
        int[][][] arr = new int[n+1][m + 1][2];

        for (int start = 1; start <= m; start++) {
            arr[n][start][1]=1;
            arr[n][start][0]=1;
        }

        for (int i = n-1; i >=0; i--) {
            for (int prev = 1; prev <= m; prev++) {
                for (int next = prev+1; next<=m; next++) {
                    arr[i][prev][1] = (arr[i][prev][1] + arr[i+1][next][0])  % 1000000007;
                }
                for (int next = 1; next<prev; next++) {
                    arr[i][prev][0] = (arr[i][prev][0] + arr[i+1][next][1])  % 1000000007;
                }
            }
        }

        int res = 0;
        for (int start = 1; start <= m; start++) {
            res = (res + arr[1][start][1]) % 1000000007;
            res = (res + arr[1][start][0])  % 1000000007 ;
        }
        return res;
    }
}
```

### Complexity
```
Time Complexity
Total states = n × m × 2
For each state, up to m transitions are explored.

Time: O(n × m²)

where m = r - l + 1
```

```
Space Complexity
DP table: O(n × m × 2)

Space: O(n × m)
```

## Approach-4: Tabulation with optimization


- Calculate the total number of available values in the range [l, r]. 
- Create a DP table where each state represents:
  - Current position in the array. 
  - Previous chosen value. 
  - Required direction (increase or decrease).
- Initialize the last position: 
  - If we are already at the last index, there is exactly one way to complete the array. 
  - Set all states at the last position to 1.
- Process positions from right to left. 
- For each position:
  - Build prefix sums for:
    - States requiring a decrease next. 
    - States requiring an increase next.
- For every previous value:
  - To find ways for an increasing move: We need the sum of all states whose values are greater than the current value. Obtain this using the prefix sum array in O(1).
  - To find ways for a decreasing move: We need the sum of all states whose values are smaller than the current value.  Obtain this using the prefix sum array in O(1).
- Store these computed values in the current DP layer. 
- After filling all layers, sum the answers corresponding to: Every possible starting value. Both starting directions. 
- Return the final count modulo 10^9 + 7.

---

### Code:
```java
class Solution {
    public int zigZagArrays(int n, int l, int r) {
        int m = r - l + 1;
        int[][][] arr = new int[n+1][m + 1][2];

        for (int start = 1; start <= m; start++) {
            arr[n][start][1]=1;
            arr[n][start][0]=1;
        }

        for (int i = n-1; i >=0; i--) {
            for (int prev = 1; prev <= m; prev++) {
                for (int next = prev+1; next<=m; next++) {
                    arr[i][prev][1] = (arr[i][prev][1] + arr[i+1][next][0])  % 1000000007;
                }
                for (int next = 1; next<prev; next++) {
                    arr[i][prev][0] = (arr[i][prev][0] + arr[i+1][next][1])  % 1000000007;
                }
            }
        }

        int res = 0;
        for (int start = 1; start <= m; start++) {
            res = (res + arr[1][start][1]) % 1000000007;
            res = (res + arr[1][start][0])  % 1000000007 ;
        }
        return res;
    }
}
```

### Complexity
```
Time Complexity
Number of DP states = n × m × 2
Each state is computed in O(1) using prefix sums.

Time Complexity: O(n × m)

where m = r - l + 1
```

```
Space Complexity
DP table stores all positions, values, and directions.

Space Complexity: O(n × m)
```
