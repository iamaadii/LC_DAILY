# Problem Statement
```
You are given an array of positive integers arr. Perform some operations (possibly none) on arr so that it satisfies these conditions:

The value of the first element in arr must be 1.
The absolute difference between any 2 adjacent elements must be less than or equal to 1. In other words, abs(arr[i] - arr[i - 1]) <= 1 for each i where 1 <= i < arr.length (0-indexed). abs(x) is the absolute value of x.
There are 2 types of operations that you can perform any number of times:

Decrease the value of any element of arr to a smaller positive integer.
Rearrange the elements of arr to be in any order.
Return the maximum possible value of an element in arr after performing the operations to satisfy the conditions.
```

### Constraints:
```
1 <= arr.length <= 10^5
1 <= arr[i] <= 10^9
```


# Approach-1: Using Direct sorting
- Sort the Array.
- Set first element as '1'.
- Initialize ans=1
- Run a loop from 1 to arr length - 1.
  - If absolute difference of current & previous element is greater than 1 set it to prevEle + 1.
  - Replace 'ans' with max of 'ans' & currEle.
- Return ans

---

### Code:
```java
class Solution {
    public int maximumElementAfterDecrementingAndRearranging(int[] arr) {
        Arrays.sort(arr);
        arr[0] = 1;
        int ans = arr[0];
        for(int i=1;i<arr.length;i++){
            if(Math.abs(arr[i]-arr[i-1])>1){
                arr[i]=arr[i-1]+1;
            }
            ans = Math.max(ans,arr[i]);
        }
        return ans;
    }
}
```

### Complexity
```
Time = O(nlogn)+O(n)
```

```
Space=O(1)
```

# Approach-2: Using count sort

## Intuition

Since we are allowed to decrement any element any number of times, values larger than n (where n is the array size) are never useful because the maximum possible answer can never exceed n. Therefore, we first clamp every element to n and store its frequency.

Now, instead of sorting the array, we process the values from 1 to n in increasing order. Suppose we have already constructed a valid array whose maximum element is ans. When processing value i, there are temp[i] occurrences available. Ideally, these occurrences can extend the constructed sequence by temp[i] positions, making the maximum value ans + temp[i]. However, since we are currently processing value i, the maximum value cannot exceed i. Hence, the new maximum element becomes:

```Math.min(i, ans + temp[i])```

By greedily updating ans for every value, we obtain the largest possible maximum element without explicitly sorting the array.

## Approach
- Let n be the length of the array. 
- Create a frequency array temp of size n + 1. 
- For every element:
  - If it is greater than n, count it as n. 
  - Otherwise, increment its corresponding frequency.
- Initialize ans = 1 since the smallest element must always be 1. 
- Traverse values from 2 to n:
  - The current value i has temp[i] occurrences. 
  - These occurrences can increase the current maximum from ans to at most ans + temp[i]. 
  - But the maximum cannot exceed i, so update:
    ```ans = Math.max(ans, Math.min(i, ans + temp[i]));```
- Return ans.


---

### Code:
```java
class Solution {
    public int maximumElementAfterDecrementingAndRearranging(int[] arr) {
        int max = arr.length;
        int[] temp = new int[max+1];
        temp[1] = 1;
        for(int e: arr){
            if(e>max){
                temp[max]+=1;
            }
            else {
                temp[e]+=1;
            }
        }

        int ans=1;
        for(int i=2;i<temp.length;i++){
            int freq = Math.min(i,ans+temp[i]);
            ans = Math.max(ans,freq);
        }
        return ans;
    }
}
```

### Complexity
```
Time:
- O(n) to count freq.
- O(n) to find answer.
```

```
Space = O(n) for frequency array
```
