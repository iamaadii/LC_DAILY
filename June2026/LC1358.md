# Problem Statement
```
Given a string s consisting only of characters a, b and c.

Return the number of substrings containing at least one occurrence of all these characters a, b and c.
```

## Constraints:
```
3 <= s.length <= 5 x 10^4
s only consists of a, b or c characters.
```


# Approach-1: 
A substring is valid if it contains at least one 'a', one 'b', and one 'c'.

Instead of generating all possible substrings (which would take O(n²) time), we use the Sliding Window (Two Pointers) technique.

The key observation is:
- As soon as the current window contains all three characters (a, b, and c), every substring obtained by extending this window to the right will also remain valid because adding more characters cannot remove an existing character. 
- If the current window is [i...j] (where j has already been included), then there are n - j + 1 valid substrings starting from index i. 
- After counting these substrings, we try to shrink the window from the left to find more valid starting positions while maintaining the condition.

This allows us to count all valid substrings efficiently without checking every possible substring.

---

## Code:
```java
class Solution {
    public int numberOfSubstrings(String s) {
        int i = 0, j=0;
        int[] freq = new int[3];
        int ans = 0;
        int  n = s.length();
        while(j<n){
            char c = s.charAt(j++);
            freq[c-'a'] += 1;
            while(freq[0]>0 && freq[1]>0 && freq[2]>0){
                ans += (n-j+1);
                freq[s.charAt(i)-'a']-=1;
                i+=1;          
            }
        }
        return ans;
    }
}
```

## Complexity
```
Time Complexity
O(2n)

Each character is added to the window once and removed once. Therefore, both pointers traverse the string at most one time.
```

```
Space Complexity
O(1)

Only a frequency array of size 3 is used, so the extra space remains constant.
```
